## 2) Table 1 dependent variable (DV): “Musical exclusiveness” = count of genres disliked (0–18)
**Construct DV as a summed count across the music-module items listed below.**

### 2.1 Underlying music taste fields (18 genre ratings)
Each is a 5-point like/dislike item with “don’t know”/missing codes. Use these data fields:

| Paper construct: genre rating | GSS field |
|---|---|
| Big band / swing | `BIGBAND` |
| Bluegrass | `BLUGRASS` |
| Country / Western | `COUNTRY` |
| Blues / R&B | `BLUES` |
| Broadway musicals / show tunes | `MUSICALS` |
| Classical / symphony & chamber | `CLASSICL` |
| Folk | `FOLK` |
| Gospel | `GOSPEL` |
| Jazz | `JAZZ` |
| Latin / Mariachi / Salsa | `LATIN` |
| Mood / easy listening | `MOODEASY` |
| New age / space | `NEWAGE` |
| Opera | `OPERA` |
| Rap | `RAP` |
| Reggae | `REGGAE` |
| Contemporary pop/rock | `CONROCK` |
| Oldies rock | `OLDIES` |
| Heavy metal | `HVYMETAL` |

### 2.2 “Disliked” indicator coding for each genre
For each genre field above, create an indicator:

- `dislike_genre = 1` if response is **4 = “dislike it”** or **5 = “dislike very much”**
- `dislike_genre = 0` if response is **1, 2, or 3**
- Treat **don’t know / missing** (all the NA-coded categories in the documentation) as **missing** for that item.

### 2.3 Scale construction rule (case exclusion for DK)
Per the analysis summary: **exclude** respondents from the DV construction if they have **any** missing / DK on **any** of the 18 genre fields.

Operationally:
- Keep only cases with non-missing values on all 18:  
  `complete.cases(BIGBAND, BLUGRASS, ..., HVYMETAL)`
- Then compute:  
  **Musical exclusiveness (DV)** = sum of the 18 `dislike_genre` indicators  
  Range: **0–18**

## 3) Table 1 independent variables (IVs): SES predictors
### 3.1 Education (years)
- **Paper variable:** Education (years)
- **GSS field:** `EDUC`
- Use numeric years as coded (0–20). Treat NA-coded values as missing.

### 3.2 Household income per capita
- **Paper variable:** Household income per capita
- **GSS fields used:** `REALINC` and `HOMPOP`
  - `REALINC` = family income in constant dollars
  - `HOMPOP` = number of persons in household
- Construct:  
  **Income per capita** = `REALINC / HOMPOP`
- Missingness:
  - If either `REALINC` or `HOMPOP` is missing/NA-coded, income-per-capita is missing.
  - If `HOMPOP` is 0 (unlikely in GSS household roster context), treat as missing/invalid to avoid division errors.

*(Note: `INCOME` is a bracketed ordinal measure; `REALINC` is the documented constant-dollar numeric income and aligns with the “per capita” construction described.)*

### 3.3 Occupational prestige
- **Paper variable:** Occupational prestige
- **GSS field:** `PRESTG80` (Respondent’s occupational prestige score, 1980)
- Use numeric prestige score; NA-coded values are missing.

## 4) Table 1 demographic / group identity covariates (Model 2 and Model 3)
### 4.1 Female indicator
- **Paper variable:** Female (1/0)
- **GSS field:** `SEX`
- Construct:
  - `female = 1` if `SEX == 2` (female)
  - `female = 0` if `SEX == 1` (male)
  - Missing if NA-coded

### 4.2 Age (continuous)
- **Paper variable:** Age
- **GSS field:** `AGE`
- Use numeric age (with 89 meaning “89 or older” as top-coded per label); NA-coded values missing.

### 4.3 Race indicators: Black / Other race (and implied White reference)
- **Paper variables:** Black, Other race (indicators); White is the omitted/reference category.
- **GSS field:** `RACE`
- Construct:
  - `black = 1` if `RACE == 2`, else 0 (when non-missing)
  - `other_race = 1` if `RACE == 3`, else 0 (when non-missing)
  - Reference/omitted: `RACE == 1` (white)

### 4.4 Hispanic indicator
- **Paper variable:** Hispanic (indicator)
- **Mapping to documentation provided:** **Not available in the attached variable list.**
  - The variable documentation excerpt does not include a Hispanic/ethnicity field (commonly `HISPANIC`, `HISPANC`, or similar in some GSS releases).
  - Therefore, within the fields shown, **Hispanic cannot be constructed**.

### 4.5 Conservative Protestant indicator
- **Paper variable:** Conservative Protestant (indicator)
- **GSS fields available in documentation:** `RELIG`, `DENOM` (and optionally `RELIG16`, `DENOM16` for “raised” measures, though Table 1 uses current identity per summary).
- Construct as a denomination-based dummy using current affiliation:
  - First restrict to Protestants: `RELIG == 1`
  - Then define **Conservative Protestant** from `DENOM` where possible. With the provided 7-category `DENOM` recode, a practical mapping is:
    - `conserv_prot = 1` if `RELIG==1` and `DENOM==1` (Baptist) **or** `DENOM==6` (Other denomination) **or** `DENOM==7` (No denomination / non-denominational church)
    - `conserv_prot = 0` otherwise (including non-Protestants and mainline-coded Protestant categories such as Methodist/Lutheran/Presbyterian/Episcopal).
- Treat NA-coded `RELIG`/`DENOM` as missing.

*(This operationalization is the closest implementable mapping given the coarse 2024-retro recode of `DENOM`. If the original analysis used a more detailed fundamentalist/mainline classification, that detail is not present in the excerpted documentation.)*

### 4.6 No religion indicator
- **Paper variable:** No religion (indicator)
- **GSS field:** `RELIG`
- Construct:
  - `no_religion = 1` if `RELIG == 4` (none)
  - `no_religion = 0` otherwise (non-missing)
  - Missing if NA-coded

### 4.7 Southern indicator
- **Paper variable:** Southern (indicator)
- **GSS field:** `REGION`
- Construct:
  - `south = 1` if `REGION == 3` (south)
  - `south = 0` if `REGION` is 1, 2, or 4
  - Missing if NA-coded

## 5) Table 1 Model 3 add-on: Political intolerance scale (0–15)
**Paper construct:** count of intolerant responses across 15 dichotomous items = (3 liberties × 5 target groups)

### 5.1 Underlying tolerance item fields (15 items)
Use these 15 GSS fields:

**Anti-religionist target (ATH):**
- Speak: `SPKATH`
- Teach: `COLATH`
- Library: `LIBATH`

**Racist / “Black genetic inferiority” target (RAC):**
- Speak: `SPKRAC`
- Teach: `COLRAC`
- Library: `LIBRAC`

**Communist target (COM):**
- Speak: `SPKCOM`
- Teach: `COLCOM`
- Library: `LIBCOM`

**Militarist target (MIL):**
- Speak: `SPKMIL`
- Teach: `COLMIL`
- Library: `LIBMIL`

**Homosexual target (HOMO):**
- Speak: `SPKHOMO`
- Teach: `COLHOMO`
- Library: `LIBHOMO`

### 5.2 “Intolerant” coding (binary) within each item
Define an intolerance indicator per item as the response that **denies** the civil liberty:

- For **SPK*** items (`SPKATH`, `SPKRAC`, `SPKCOM`, `SPKMIL`, `SPKHOMO`):
  - intolerant = 1 if value **2 = “not allowed”**
  - intolerant = 0 if value **1 = “yes, allowed to speak”**

- For **COL*** items (`COLATH`, `COLRAC`, `COLCOM`, `COLMIL`, `COLHOMO`):
  - intolerant = 1 if value **5 = “not allowed”** (or **“not fired”** is coded 5 only for `COLCOM`; intolerance corresponds to *firing*, which is **4** there—see next bullet)
  - Special case `COLCOM` (communist teacher):
    - intolerant = 1 if **4 = “yes, fired”**
    - intolerant = 0 if **5 = “not fired”**

- For **LIB*** items (`LIBATH`, `LIBRAC`, `LIBCOM`, `LIBMIL`, `LIBHOMO`):
  - intolerant = 1 if value **1 = “remove”**
  - intolerant = 0 if value **2 = “not remove”**

Treat NA-coded values (dk/na/refused/etc.) as missing.

### 5.3 Scale construction
- **Political intolerance scale** = sum of the 15 intolerance indicators (range 0–15)
- For inclusion in Model 3, require non-missing on all 15 items (or otherwise follow the paper’s approach; the summary implies a complete count across 15 items).
- **Split-ballot linkage:** use `BALLOT` to reflect that these items appear for only a subset; Model 3’s N is smaller because only some ballots include the tolerance battery.
