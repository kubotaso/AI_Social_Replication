## Mapping of Table 3 analysis variables to GSS data fields (per provided documentation)

### 1) Dataset restriction / selection
- **Target survey year (GSS 1993)** → Use **`YEAR`**
  - **Filter:** `YEAR == 1993`
  - This ensures all frequencies/means are computed for the 1993 GSS sample only, matching the table title.

- **Respondent identifier (not displayed in Table 3, but used for row-level uniqueness if needed)** → **`ID`**
  - No filtering; used only as an optional record key.

---

### 2) Music-genre attitude items (Table 3 “columns”)
Each *music genre* in the table corresponds to one GSS variable that records the respondent’s attitude toward that genre. The table is produced by generating a separate frequency distribution (and mean) for each of the following variables:

1. **Latin/Salsa** → **`LATIN`** (Latin/Mariachi/Salsa)  
2. **Jazz** → **`JAZZ`**  
3. **Blues/R&B** → **`BLUES`** (Blues or Rhythm and Blues)  
4. **Show Tunes** → **`MUSICALS`** (Broadway musicals/show tunes)  
5. **Oldies** → **`OLDIES`** (Oldies rock)  
6. **Classical/Chamber** → **`CLASSICL`** (Classical music—symphony & chamber)  
7. **Reggae** → **`REGGAE`**  
8. **Swing/Big Band** → **`BIGBAND`** (Big Band/Swing)  
9. **New Age/Space** → **`NEWAGE`** (New age/space music)  
10. **Opera** → **`OPERA`**  
11. **Bluegrass** → **`BLUGRASS`**  
12. **Folk** → **`FOLK`**  
13. **Pop/Easy Listening** → **`MOODEASY`** (Mood/easy listening)  
14. **Contemporary Rock** → **`CONROCK`** (Contemporary pop/rock)  
15. **Rap** → **`RAP`**  
16. **Heavy Metal** → **`HVYMETAL`**  
17. **Country/Western** → **`COUNTRY`**  
18. **Gospel** → **`GOSPEL`**

These 18 variables are the “analysis variables” being summarized to create Table 3.

---

### 3) Response categories (Table 3 “rows”) and how they map to coded values
For **each** of the 18 genre variables above, the frequency table rows map directly to the value labels in the documentation:

**Attitude (non-missing categories; displayed as (1)–(5) in the table):**
- **(1) Like very much** → value **`1`**
- **(2) Like it** → value **`2`**
- **(3) Mixed feelings** → value **`3`**
- **(4) Dislike it** → value **`4`**
- **(5) Dislike very much** → value **`5`**

**Missing-type categories (displayed with “(M)” in the table):**
- **(M) Don’t know much about it** → **`NA(d)`** (documentation label: “don’t know”)
  - For these music items, `NA(d)` corresponds to “don’t know much about it” per the question wording.
- **(M) No answer** → **`NA(n)`** (documentation label: “no answer”)

**Other non-substantive NA codes present in the documentation** (e.g., `NA(i)` iap, `NA(r)` refused, `NA(s)` skipped on web, etc.) are treated as missing values in general; Table 3, per the summary, specifically displays the “Don’t know much about it” and “No answer” missing categories.

---

### 4) Mean construction (Table 3 “Mean” row/column statistic)
For each genre variable (e.g., `LATIN`, `JAZZ`, …):

- **Mean attitude score** is computed on the **1–5 scale** using only valid attitude responses:
  - Include: values **1, 2, 3, 4, 5**
  - Exclude from mean (treat as missing): **`NA(d)`** (“don’t know much about it”) and **`NA(n)`** (“no answer”) (and any other NA-coded values if present)

So, for every genre variable `GENRE_VAR` in the list above:

- **Mean(GENRE_VAR)** = average of `GENRE_VAR` over cases where `GENRE_VAR ∈ {1,2,3,4,5}`.

---

### 5) Output structure mapping to the table layout
- **Table columns** → the 18 genre variables listed in Section 2 (each variable becomes one genre column).
- **Table rows** → frequency counts for:
  - values 1–5 (attitude categories),
  - plus counts for `NA(d)` (don’t know much about it) and `NA(n)` (no answer),
  - plus the mean of 1–5 (with NA categories excluded).

- **Displayed statistics**:
  - **Counts only** (no percentages), per the analysis summary.
