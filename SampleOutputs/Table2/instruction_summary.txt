## Mapping of Table 2 (Bryson 1996) analysis variables to the provided GSS data fields

### Dataset / inclusion
- **Data source / year restriction:** use **1993 GSS respondents only**
  - **Field:** `YEAR`
  - **Filter:** `YEAR == 1993`
- **Unit of analysis:** individual respondent
  - **Identifier field:** `ID`

---

## Dependent variables (two count outcomes)

### DV1: “Dislike of minority-linked genres” (count of 6)
**Concept (analysis):** Count how many of the following 6 genres the respondent **dislikes** (either “dislike it” or “dislike very much”).

**Input fields (music ratings):**
- Rap → `RAP`
- Reggae → `REGGAE`
- Blues / R&B → `BLUES`
- Jazz → `JAZZ`
- Gospel → `GOSPEL`
- Latin → `LATIN`

**Dislike indicator coding (per genre):**
- Set `dislike_genre = 1` if field value is **4** (“dislike it”) or **5** (“dislike very much”)
- Set `dislike_genre = 0` if field value is **1**, **2**, or **3**
- Treat GSS missing codes (`NA(...)` including “don’t know / not available / refused / etc.”) as **missing** for that genre

**DV1 construction:**
- `DV1 = RAP_dislike + REGGAE_dislike + BLUES_dislike + JAZZ_dislike + GOSPEL_dislike + LATIN_dislike`
- If any of the six inputs are missing, DV1 will be missing (consistent with listwise handling described in the summary).

---

### DV2: “Dislike of remaining genres” (count of 12)
**Concept (analysis):** Count dislikes among the **other 12** music genres on the 1993 module (i.e., all music items except the six used in DV1).

**Input fields (remaining music ratings):**
- Big band/swing → `BIGBAND`
- Bluegrass → `BLUGRASS`
- Country/Western → `COUNTRY`
- Broadway musicals/show tunes → `MUSICALS`
- Classical → `CLASSICL`
- Folk → `FOLK`
- Mood/easy listening → `MOODEASY`
- New age/space → `NEWAGE`
- Opera → `OPERA`
- Contemporary pop/rock → `CONROCK`
- Oldies rock → `OLDIES`
- Heavy metal → `HVYMETAL`

**Dislike indicator coding (per genre):** same as DV1
- 1 if **4/5**, 0 if **1/2/3**, missing if `NA(...)`

**DV2 construction:**
- `DV2 = sum(dislike indicators across the 12 fields above)`
- If any of the 12 inputs are missing, DV2 will be missing (matching the described listwise construction).

---

## Key independent variable

### Racism score (0–5 scale)
**Concept (analysis):** Sum of 5 dichotomous racial-attitude items, coded so higher = more racist sentiment, producing a 0–5 scale.

**Fields corresponding to the 5 items listed in the summary:**
1) Objection to sending children to a school where > half Black  
   - **Field:** `RACHAF`
2) Favor/oppose busing  
   - **Field:** `BUSING`
3) Attribute inequality mainly to discrimination  
   - **Field:** `RACDIF1`
4) Attribute inequality to lack of educational opportunity  
   - **Field:** `RACDIF3`
5) Attribute inequality to lack of motivation/will power  
   - **Field:** `RACDIF4`

**Dichotomous “racist-endorsing” direction (as implied by the item wordings):**
- `RACHAF`: **1 = yes, object** → score 1; **2 = no** → score 0
- `BUSING`: **2 = oppose** → score 1; **1 = favor** → score 0
- `RACDIF1` (discrimination): **2 = no** (i.e., not mainly due to discrimination) → score 1; **1 = yes** → score 0
- `RACDIF3` (chance for education): **2 = no** → score 1; **1 = yes** → score 0
- `RACDIF4` (motivation/will power): **1 = yes** → score 1; **2 = no** → score 0

**Scale construction:**
- `RACISM_SCORE = RACHAF_r + BUSING_r + RACDIF1_r + RACDIF3_r + RACDIF4_r`
- Range: 0–5 if all five components are present
- Any `NA(...)` on component items → component missing → scale missing (listwise in regression)

## Socioeconomic-status controls

### Education (years)
- **Analysis variable:** Education (years)
- **Field:** `EDUC`
- **Use as:** continuous years of schooling completed (0–20 as labeled)

### Household income per capita
- **Analysis variable:** “Household income per capita”
- **Fields used to compute:**
  - Family income (constant dollars) → `REALINC`
  - Household size (# persons) → `HOMPOP`
- **Construction:**
  - `INCOME_PC = REALINC / HOMPOP`
  - If `REALINC` or `HOMPOP` missing (or HOMPOP==0, if present), then `INCOME_PC` missing

*(Note: `INCOME` is categorical; `REALINC` is the appropriate numeric field in the provided documentation for a per-capita computation.)*

### Occupational prestige
- **Analysis variable:** Occupational prestige
- **Field:** `PRESTG80`

## Demographic / group-identity controls

### Female (indicator)
- **Analysis variable:** Female
- **Field:** `SEX`
- **Coding:**
  - `FEMALE = 1` if `SEX == 2` (female)
  - `FEMALE = 0` if `SEX == 1` (male)

### Age (years)
- **Analysis variable:** Age
- **Field:** `AGE`
- **Use as:** continuous age in years (with 89 meaning 89+ as per label)

### Race / ethnicity indicators (reference = non-Hispanic White)
The documentation provided includes `RACE` but does **not** include a separate Hispanic ethnicity field. Map what is available as follows:

- **Black indicator**
  - **Field:** `RACE`
  - **Coding:** `BLACK = 1` if `RACE == 2`, else `0`
- **Other race indicator**
  - **Field:** `RACE`
  - **Coding:** `OTHER_RACE = 1` if `RACE == 3`, else `0`
- **White (reference category for race dummies)**
  - **Field:** `RACE`
  - **Condition:** `RACE == 1` and not otherwise flagged by ethnicity (but see Hispanic note below)

**Hispanic indicator (as specified in analysis summary):**
- **Analysis variable:** Hispanic (indicator)
- **Field in provided documentation:** **not present**
  - No direct mapping is available from the attached fields. (Variables like `ETHNIC` reflect ancestry origins, not a clean Hispanic ethnicity indicator equivalent to the typical GSS `HISPANIC`/`HISPANICx` flag.)

### Conservative Protestant (indicator)
- **Analysis variable:** Conservative Protestant
- **Fields available:** `RELIG`, `DENOM` (broad 7-category recode)
- **Mapping in this documentation context:** Use a Protestant-based denomination proxy:
  - Base Protestant identification: `RELIG == 1` (protestant)
  - **Conservative Protestant indicator (approximate with available fields):**
    - `CONS_PROT = 1` if `RELIG == 1` and `DENOM == 1` (baptist)
    - `CONS_PROT = 0` otherwise
  - *(This uses the only clearly “conservative Protestant”-aligned denomination explicitly available in the provided broad `DENOM` recode.)*

### No religion (indicator)
- **Analysis variable:** No religion
- **Field:** `RELIG`
- **Coding:** `NO_RELIGION = 1` if `RELIG == 4` (none), else `0`

### Southern (indicator)
- **Analysis variable:** Southern
- **Field:** `REGION`
- **Coding:** `SOUTHERN = 1` if `REGION == 3` (south), else `0`

## Regression specification linkage (for running Table 2 models)
Use the same RHS predictors for both models:

- **Model 2A DV:** `DV1` (minority-linked dislike count)
- **Model 2B DV:** `DV2` (remaining-genres dislike count)

**Predictor field mapping summary:**
- Racism score (0–5) → constructed from `RACHAF`, `BUSING`, `RACDIF1`, `RACDIF3`, `RACDIF4`
- Education → `EDUC`
- Income per capita → `REALINC` / `HOMPOP`
- Occupational prestige → `PRESTG80`
- Female → `SEX` (2=female)
- Age → `AGE`
- Black → `RACE` (2)
- Hispanic → **no field provided**
- Other race → `RACE` (3)
- Conservative Protestant → `RELIG` + `DENOM` (as defined above)
- No religion → `RELIG` (4)
- Southern → `REGION` (3)

**Missing data handling (as described):**
- Treat all `NA(...)` codes as missing
- Run each model with listwise deletion on the DV and all included predictors (which will also yield different N across the two DVs due to differing item availability/missingness).
